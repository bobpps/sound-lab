# Code Review: feat/19-frontend-deps-tailwind

## Потенциальные баги

### [BUG-1] Удалены CSS-стили, но HTML-разметка осталась нетронутой (Major)
**Файл:** `frontend/src/App.tsx:12-116`, `frontend/src/App.css` (удален), `frontend/src/index.css:1`
**Что не так:** Удалены `App.css` (184 строки) и полностью заменен `index.css` (111 строк переменных, типографики и layout-стилей) на одну строку `@import "tailwindcss"`. При этом `App.tsx` продолжает использовать CSS-классы из удалённых файлов: `.hero`, `.base`, `.framework`, `.vite`, `.counter`, `.ticks`, `.icon`, `.logo`, `.button-icon`, а также ID-селекторы `#center`, `#next-steps`, `#docs`, `#social`, `#spacer`, `#root`.
**Почему это плохо:** Приложение полностью потеряло все стили. Ни один из этих классов не является Tailwind-утилитой. Результат -- каша из нестилизованных элементов на экране. Boilerplate Vite-страница, конечно, не production-код, но оставлять явно сломанную разметку с мёртвыми классами -- значит маскировать проблему. Либо надо было удалить и сам boilerplate-компонент, либо переписать его на Tailwind-классы.

## Архитектурные замечания

### [ARCH-1] Proxy rewrite маскирует отсутствие API-префикса на бэкенде (Minor)
**Файл:** `frontend/vite.config.ts:8-13`
**Что не так:** Proxy настроен так, что `/api/dialogs` -> `http://localhost:3000/dialogs` (strip `/api`). Бэкенд использует `@fastify/autoload` с `dirNameRoutePrefix: true` -- маршруты монтируются напрямую как `/dialogs`, `/providers` и т.д. без какого-либо общего `/api` префикса.
**Почему это плохо:** Само по себе это рабочее решение и соответствует `frontend/CLAUDE.md` ("`server.proxy: { '/api': { target: 'http://localhost:3000' } }`"). Однако в production-деплое (без Vite dev server) потребуется аналогичный rewrite на уровне nginx/reverse-proxy, и это нигде не задокументировано. Также отсутствует `changeOrigin: true` -- на localhost неважно, но привычка опускать его ведёт к багам при проксировании на другие хосты.

## Качество кода

### [QUAL-1] Мёртвый код в App.tsx (Major)
**Файл:** `frontend/src/App.tsx:1-121`
**Что не так:** Весь компонент `App` -- это Vite boilerplate: счётчик кликов, ссылки на Vite/React доку, Discord и Bluesky. Ни одна строка этого кода не имеет отношения к Sound Lab. Импорты `reactLogo`, `viteLogo`, `heroImg`, `useState` -- всё мёртвое для реального приложения.
**Почему это плохо:** Этот PR устанавливает зависимости для реального фронтенда (React Router, TanStack Query, Zustand, React Hook Form, Zod), но оставляет на месте весь boilerplate. Следующему PR придётся всё равно удалять это. Установка серьёзного стека поверх демо-заглушки выглядит как незавершённая работа.

### [QUAL-2] Zod v4 -- проверка совместимости с @hookform/resolvers (Minor)
**Файл:** `frontend/package.json:13,20`
**Что не так:** Установлен `zod: ^4.3.6` (Zod v4, мажорный rewrite с новым API) вместе с `@hookform/resolvers: ^5.2.2`. Zod v4 требует использования `@hookform/resolvers/zod/v4` вместо `@hookform/resolvers/zod`.
**Почему это плохо:** Если импортировать resolver стандартным путём (`@hookform/resolvers/zod`), формы будут падать в runtime. Это не баг прямо сейчас (resolver нигде не используется), но это мина для следующего разработчика, который не проверит совместимость и импортирует по привычке.
